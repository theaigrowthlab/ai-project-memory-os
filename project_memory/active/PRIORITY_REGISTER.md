# Priority Register

**Purpose:** Scored, rankable backlog. Complements (does not replace) your main task list / issue tracker.

**Owner:** PM maintains scores · Founder/PO re-ranks · All roles can request a re-score

---

## Scoring Rubric

Score each item 1-5 on:

| Dimension | 1 | 3 | 5 |
|---|---|---|---|
| **Revenue impact** | None / indirect | Supports revenue | Directly generates revenue |
| **User impact** | Nice-to-have | Noticeable improvement | Unblocks critical user path |
| **Effort** | 5 = trivial … 1 = weeks | | |
| **Risk if deferred** | None | Annoying | Blocks other work / security issue |
| **Strategic fit** | Off-roadmap | Neutral | Core to current phase |

**Total score = Revenue + User + Effort + Risk + Strategic** (max 25)

Scores are **advisory only** — Founder/PO makes final priority calls.
